oldfashion_orw

代码

整个流程非常简单，根据输入的 size 大小输入 content ，要求 size 不能超过 32 ，nbytes 是 int64 类型，所以想到用负数绕过，从而栈溢出

其中有个 disable_syscall()

工具查一下

execve 被禁，所以要用 orw，并且此时 openat 也被禁了，而 glibc 中的 open 函数是调用了 opennat 的，所以直接调用会 Bad system call ，这时得用 syscall;ret 调用。

ROPgadget 搜出来的 gadget 是没有 ret 的，所以这时要用 opcode 实现

首先可以用 pwntools 模块里的 asm 获得 syscall;ret的 opcode

接着直接搜索

并且题目给了一个 start.sh 文件，可以看出 flag 文件名后面还跟了 20 个随机数

所以单纯的 orw 是不够的，还需要找到 flag 的文件名

这时可以通过 open 一个 file，然后用 getdents64 系统调用读取目录，getdents64 会返回一个结构体

只要能 write 出这个结构体的 d_name[ ] 就可以获得 flag 了。

说一下 open 和 getdents64 的实现过程

  1. open打开当前目录的句柄，这个句柄对应内核中一个file结构。
    file结构中的 file_operations 函数集从 inode 结构中获得，就是 ext2_file_operations
  2. getdents64 调用 file->f_op->readdir() 实际上是调用了 ext2_file_operations 中的 readdir()，
    由ext2文件系统驱动读取当前目录下面的文件项。

具体代码实现参考:http://www.cppblog.com/momoxiao/archive/2010/04/04/111594.html
